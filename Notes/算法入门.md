# 2、大O表示法

算法的输入大小和运步骤之间的关系叫算法的复杂度，表示运行速度的量级

算法复杂度 叫大O表示法

算法复杂度 O（N^2）效率不高，随着数组的增大，对效率的影响越来越大

# 3、归并排序

分组

算法复杂度是O（n^log n）

n是需要 比较+合并的次数，与数组大小成正比

log n 是合并步骤的次数

# 4、Dikstra算法

总是从成本最低的节点开始

第一步先把分支都跑一遍计算跑到所有节点所需要的成本，然后重复接下来的操作：找到最小的然后跑一下下个分支，未到终点就继续跑一下最小的的下个分支 这样循环

算法复杂度是O（n^2）
